الأولوية: حرجة - يعالج جذر 100% من أخطاء النسب المالية
1. الملخص التنفيذي
التحقيق الميداني في ملفات AAPL وJPM وTSLA كشف أن البيانات المجمعة من SEC صحيحة بنسبة 100%. كل الأرقام الرسمية مطابقة تماماً لتقارير 10-K المنشورة. المشكلة الوحيدة هي أن كل بند مالي رئيسي موجود في Layer1_Raw_SEC مرتين: مرة بالمليون دولار (صحيح)، ومرة بالدولار الكامل (خاطئ مليون مرة).
CODEX لا يملك حالياً قاموس أولويات (Priority Dictionary) يحدد أيّ Label يُختار عند وجود تكرار. هذا الغياب هو الجذر الوحيد لكل الأخطاء السابقة: ROE بالنانو، Cost_of_Debt معدوم، EV/EBITDA = NaN، وفشل ميزان الاعتماد.

3. آلية الاكتشاف التلقائي للوحدة
CODEX يجب أن يطبق الخوارزمية التالية على كل بند قبل استخدامه في أي حساب:
خوارزمية تطبيع الوحدة (Unit Normalization Algorithm)
def detect_and_normalize(value, context_median):
    """
    context_median = الوسيط للبنود الأخرى في نفس الشركة/السنة
    """
    if value <= 0:
        return value  # لا تطبيع للصفر أو السالب
    ratio = value / context_median
    if ratio > 500_000:  # كبير مليون مرة من المعدل
        return value / 1_000_000  # تحويل من دولار إلى مليون
    elif ratio > 500:  # كبير ألف مرة من المعدل
        return value / 1_000  # تحويل من ألف إلى مليون
    else:
        return value  # الوحدة صحيحة
قاعدة القرار عند وجود نسختين بنفس المعنى:
# إذا وُجد label_A و label_B يمثلان نفس المفهوم:
# اختر دائماً الأصغر (بالمليون)
def pick_correct_twin(val_A, val_B):
    if val_A > 0 and val_B > 0:
        ratio = max(val_A, val_B) / min(val_A, val_B)
        if ratio > 500_000:  # نفس الرقم × مليون
            return min(val_A, val_B)  # اختر الأصغر دائماً
    return val_A  # fallback

5. خوارزمية اختيار Label المُحدَّثة (CODEX v3)
CANONICAL_LABEL_PRIORITY = {
    # === قائمة الدخل ===
    'Revenue': {
        'preferred': ['Revenues', 'Revenue', 'Net sales',
                      'Net revenue', 'Total revenues'],
        'skip': ['SalesRevenueNet', 'RevenueFromContractWith...'],
        'skip_condition': lambda v: v > 1_000_000_000
    },
    'COGS': {
        'preferred': ['CostOfRevenue', 'Cost of Revenue',
                      'Cost of revenues', 'COGS'],
        'skip': ['CostOfGoodsAndServicesSold', 'CostOfSales'],
        'skip_condition': lambda v: v > 1_000_000_000
    },
    'NetIncome': {
        'preferred': ['NetIncomeLoss', 'NetIncome', 'Net income'],
        'skip': ['ProfitLoss'],
        'skip_condition': lambda v: v > 10_000_000_000
    },
    # === الميزانية العمومية ===
    'Equity': {
        'preferred': ['StockholdersEquity', 'Total Equity',
                      "Stockholders' equity"],
        'skip': ['LiabilitiesAndStockholdersEquity',
                 'TotalEquity (if == Assets)'],
        'skip_always': ['LiabilitiesAndStockholdersEquity']
    },
    'Assets': {
        'preferred': ['Assets', 'TotalAssets'],
        'normalize': lambda v: v/1_000_000 if v > 1_000_000_000 else v
    },
    # === الفائدة ===
    'InterestExpense': {
        'preferred': ['InterestExpense', 'Interest Expense',
                      'InterestExpense_Hierarchy'],
        'skip': ['InterestCostsIncurred', 'InterestPaidNet'],
        'skip_condition': lambda v: v > 1_000_000_000
    },
    # === الاستثمار الرأسمالي ===
    'CapEx': {
        'preferred': ['PaymentsToAcquirePropertyPlantAndEquipment'],
        'normalize': lambda v: v/1_000_000 if v > 1_000_000_000 else v,
        'skip': ['CapitalExpenditures'],  # هذا بالدولار
    },
}
def get_canonical_value(concept_key, raw_sec_data, year):
    config = CANONICAL_LABEL_PRIORITY[concept_key]
    candidates = {}
    for label in config['preferred']:
        val = raw_sec_data.get(label, {}).get(year)
        if val is not None and val > 0:
            # تطبيع الوحدة إذا لزم
            if 'normalize' in config:
                val = config['normalize'](val)
            elif val > 1_000_000_000:
                val = val / 1_000_000  # تلقائي
            candidates[label] = val
    if not candidates:
        raise MissingDataError(f'{concept_key} not found for {year}')
    # اختر أول قيمة من القائمة المفضلة
    return candidates[list(candidates.keys())[0]]

VALIDATION_RULES = {
    'ROE':           {'min': -0.5,  'max': 5.0,   'action': 'HALT'},
    'cost_of_debt':  {'min': 0.005, 'max': 0.30,  'action': 'HALT'},
    'ev_ebitda':     {'min': 0.5,   'max': 150.0, 'action': 'HALT'},
    'gross_margin':  {'min': -0.5,  'max': 1.0,   'action': 'HALT'},
    'asset_turnover':{'min': 0.01,  'max': 10.0,  'action': 'HALT'},
    'fair_value':    {'min_ratio': 0.1, 'max_ratio': 10.0, 'action': 'FLAG'},
}
def validate_ratio(name, value, current_price=None):
    rule = VALIDATION_RULES.get(name, {})
    if not rule: return value
    if 'min' in rule and (value < rule['min'] or value > rule['max']):
        if rule['action'] == 'HALT':
            raise UnitMismatchError(
                f'{name}={value:.6f} خارج النطاق [{rule["min"]},{rule["max"]}]'
                f' → خطأ وحدة محتمل'
            )

8. خطة التنفيذ المرحلية
المرحلة الأولى: إضافة القاموس (يوم واحد)
إضافة CANONICAL_LABEL_PRIORITY كملف constants.py منفصل
تعديل دالة get_raw_value() لتستدعي القاموس أولاً قبل أي extraction
إضافة detect_and_normalize() كطبقة وسيطة إلزامية
المرحلة الثانية: إضافة Validation Gate (يوم واحد)
إضافة validate_ratio() بعد كل حساب مباشرة
تحويل HALT إلى exception يوقف الحساب ويسجل UNIT_ERROR
إضافة عمود data_source في كل ورقة (Canonical_Label_Used)
المرحلة الثالثة: اختبار الرجعي (يوم واحد)
إعادة تشغيل AAPL/JPM/TSLA والتحقق من جدول الأثر (القسم 7)
قبول النتيجة فقط إذا: ROE و Cost_of_Debt ضمن النطاق المقبول
تحقق من Balance Sheet = 0 فجوة لكل سنوات TSLA

10. ملاحظة ختامية
هذا الإصلاح لا يتطلب تغيير منطق الحساب ولا تحديث النماذج. البيانات صحيحة 100%. المطلوب فقط: قاموس أولويات + كاشف وحدات + بوابة تحقق. هذه التعديلات الثلاثة تحل 100% من الأخطاء الموثقة في التقارير السابقة.
