## 📊 المؤشرات الخمسة

### 1️⃣ **مؤشر احتمالية الاحتيال (AI Fraud Probability)** 🚨

#### **الهدف:**
كشف التجميل المالي والتلاعب في القوائم المالية

#### **المنطق:**
يحلل العلاقة بين الأرباح المحاسبية والتدفقات النقدية الفعلية. إذا كانت الشركة تُظهر أرباحاً عالية لكن التدفقات النقدية منخفضة، فهذا إشارة احتيال محتملة.

#### **البيانات المستخدمة:**
- `Accruals_Ratio` - الفجوة بين الأرباح والتدفقات
- `Net Profit Margin` - هامش الربح الصافي
- `Operating CF` - التدفق النقدي التشغيلي

#### **المخرجات:**
```
✅ احتمالية الاحتيال: 15.2%
✅ عدد العلامات الحمراء: 2
✅ مستوى المخاطر: منخفض
✅ التوصية: آمن
```

#### **علامات حمراء:**
1. **Accruals عالية:** `abs(accruals) > 8%` → زيادة في الأرباح المحاسبية دون دعم نقدي
2. **هامش ربح عالي مع تدفق سلبي:** `margin > 10%` لكن `OCF < 0` → تناقض خطير
3. **فجوة كبيرة بين NI و OCF:** `abs((NI - OCF) / NI) > 30%` → تباين غير مبرر

#### **التقييم:**
- **< 20%:** منخفض ✅ آمن
- **20-50%:** متوسط ⚠️ مراجعة دقيقة
- **> 50%:** مرتفع 🚨 تجنب

#### **مثال عملي:**
```
شركة X:
- صافي الدخل: $500M
- التدفق التشغيلي: $150M
- Accruals Ratio: 0.12 (عالي جداً!)

النتيجة:
🚨 احتمالية احتيال: 65%
🚨 علامات حمراء: 3
❌ التوصية: تجنب - فجوة كبيرة بين الأرباح والنقد
```

### 2️⃣ **التنبؤ بالتعثر المتقدم (Dynamic Failure Prediction)** 📉

#### **الهدف:**
التنبؤ باحتمالية تعثر الشركة خلال 3-5 سنوات قادمة

#### **المنطق:**
بدلاً من رقم ثابت (مثل Altman Z-Score)، يحلل **الاتجاهات** ويعطي احتمالية مئوية ديناميكية

#### **البيانات المستخدمة:**
- `Altman_Z_Score` (آخر 5 سنوات) - مؤشر الإفلاس
- `Net_Debt_EBITDA` (الاتجاه) - نسبة الديون للأرباح
- `Interest Coverage Ratio` - قدرة دفع الفوائد

#### **المخرجات:**
```
✅ احتمالية التعثر خلال 3 سنوات: 12.5%
✅ احتمالية التعثر خلال 5 سنوات: 18.3%
✅ مستوى المخاطر: منخفض
✅ المخاوف الرئيسية: لا توجد مخاوف كبيرة
```

#### **العوامل المؤثرة:**

**1. اتجاه Z-Score:**
```python
if z_trend < -0.5:  # تدهور
    failure_score += 0.25

if z_latest < 1.80:  # منطقة خطر
    failure_score += 0.30
```

**2. نسبة الديون:**
```python
if avg_debt_ebitda > 5:
    failure_score += 0.30  # ديون عالية جداً
elif avg_debt_ebitda > 3:
    failure_score += 0.15  # ديون مرتفعة
```

**3. تغطية الفوائد:**
```python
if avg_interest_coverage < 1.5:
    failure_score += 0.25  # خطر عدم السداد
elif avg_interest_coverage < 3:
    failure_score += 0.10
```

#### **التقييم:**
- **< 15%:** منخفض ✅
- **15-40%:** متوسط ⚠️
- **> 40%:** مرتفع 🚨

#### **مثال عملي:**
```
شركة Y:
- Z-Score 2023: 2.8
- Z-Score 2024: 2.2
- Z-Score 2025: 1.6 (⬇️ تدهور)
- Net Debt/EBITDA: 6.5 (عالي!)
- Interest Coverage: 1.3 (منخفض!)

النتيجة:
🚨 احتمالية التعثر (3 سنوات): 48%
🚨 احتمالية التعثر (5 سنوات): 62%
⚠️ المخاوف: Z-Score منخفض، ديون مرتفعة، صعوبة في تغطية الفوائد
```

### 3️⃣ **درجة استدامة النمو (Growth Sustainability Grade)** 📈

#### **الهدف:**
تقييم قدرة الشركة على تحقيق النمو المستدام بدون زيادة خطيرة في الديون

#### **المنطق:**
يقيس ما إذا كان النمو الحالي/المتوقع مدعوماً بـ ROIC قوي و Retention عالي، أم يتطلب اقتراض سيضر بالتصنيف الائتماني

#### **البيانات المستخدمة:**
- `SGR_Internal` - معدل النمو المستدام
- `ROIC` - العائد على رأس المال
- `Retention_Ratio` - نسبة الأرباح المحتفظ بها
- `Actual NI Growth` - النمو الفعلي

#### **المخرجات:**
```
✅ النتيجة: 78 / 100
✅ الدرجة: B
✅ التقييم: نمو مستدام
✅ لا توجد مخاوف من الديون
```

#### **معايير التقييم:**

**1. ROIC (30 نقطة):**
```python
if roic > 15%:
    score += 30  # ممتاز
elif roic > 10%:
    score += 20  # جيد
else:
    score += 10  # ضعيف
```

**2. Retention Ratio (25 نقطة):**
```python
if retention > 70%:
    score += 25  # توزع أقل = إعادة استثمار أكثر
elif retention > 50%:
    score += 15
```

**3. توافق النمو مع SGR (30 نقطة):**
```python
if actual_growth <= sgr:
    score += 30  # النمو ضمن الحدود المستدامة
elif actual_growth <= sgr * 1.5:
    score += 15  # نمو مقبول
```

**4. تحذير الديون (-20 نقطة):**
```python
debt_growth = (debt_2025 - debt_2024) / debt_2024
if debt_growth > actual_ni_growth * 1.5:
    score -= 20  # الديون تنمو أسرع من الأرباح!
```

#### **الدرجات:**
- **A (85-100):** نمو مستدام ممتاز ✅
- **B (60-84):** نمو مستدام جيد ✅
- **C (40-59):** نمو محفوف بالمخاطر ⚠️
- **D (< 40):** نمو غير مستدام 🚨

#### **مثال عملي:**
```
شركة Z:
- ROIC: 18% (✅ ممتاز → 30 نقطة)
- Retention: 75% (✅ عالي → 25 نقطة)
- SGR: 12%
- Actual Growth: 15% (⚠️ أعلى من SGR → 15 نقطة)
- Debt Growth: 25% vs NI Growth 15% (🚨 → -20 نقطة)

النتيجة: 50 / 100
الدرجة: C
⚠️ تحذير: النمو يتطلب ديون كثيرة!
```

### 4️⃣ **تحليل كفاءة رأس المال العامل الذكي (AI Working Capital Analysis)** 💰

#### **الهدف:**
التنبؤ بأزمات السيولة النقدية رغم وجود أرباح

#### **المنطق:**
يحلل دورة التحويل النقدي (CCC) واتجاهها. إذا كان CCC يزداد مع نمو سريع في المبيعات، فقد تحدث أزمة سيولة.

#### **البيانات المستخدمة:**
- `CCC_Days` (آخر 5 سنوات) - دورة التحويل النقدي
- `Inventory Days` - فترة بقاء المخزون
- `AR Days` - فترة التحصيل
- `Revenue Growth` - نمو المبيعات

#### **المخرجات:**
```
✅ احتمالية أزمة سيولة: 18.5%
✅ دورة التحويل النقدي: 65.3 يوم
✅ الاتجاه: ⬆️ تزايد بمعدل 4.2 يوم/سنة
⚠️ التوصية: مراقبة
```

#### **العوامل المؤثرة:**

**1. CCC مرتفع:**
```python
if ccc > 90:
    risk += 0.30  # بطيء جداً
elif ccc > 60:
    risk += 0.15
```

**2. اتجاه تصاعدي:**
```python
if ccc_trend > 5:  # يزداد بـ5+ يوم/سنة
    risk += 0.25  # تدهور!
```

**3. نمو سريع + CCC عالي:**
```python
if revenue_growth > 15% and ccc > 60:
    risk += 0.20  # نمو سريع يحتاج مال عامل كثير!
```

#### **التقييم:**
- **< 20%:** منخفض ✅ جيد
- **20-50%:** متوسط ⚠️ مراقبة
- **> 50%:** مرتفع 🚨 تحسين عاجل

#### **مثال عملي:**
```
شركة W:
- CCC: 95 يوم (⬆️ عالي → 0.30)
- Trend: +7 يوم/سنة (⬆️ تدهور → 0.25)
- Revenue Growth: 20% (🚀 سريع → 0.20)

النتيجة:
🚨 احتمالية أزمة سيولة: 75%
❌ التوصية: تحسين عاجل
⚠️ المشكلة: نمو سريع لكن رأس المال العامل بطيء جداً!
```

### 5️⃣ **تقييم جودة الاستثمار النهائي (AI Investment Quality Score)** ⭐

#### **الهدف:**
إعطاء حكم نهائي شامل على جودة الاستثمار

#### **المنطق:**
يدمج 5 مقاييس رئيسية في نتيجة واحدة (0-100) ويعطي حكماً نهائياً

#### **البيانات المستخدمة:**
- `Economic_Spread` - ROIC - WACC
- `FCF_Yield` - عائد التدفق النقدي الحر
- `Investment_Score` - النتيجة الاستثمارية
- `ROIC` - العائد على رأس المال
- `Altman Z-Score` - الأمان المالي

#### **المخرجات:**
```
✅ النتيجة النهائية: 87 / 100
✅ الحكم: جوهرة مخفية 💎
✅ التوصية الاستثمارية: شراء قوي
✅ أفضل من: 95% من الشركات
```

#### **معايير التقييم:**

**1. Economic Spread (25 نقطة):**
```python
if economic_spread > 10%:
    score += 25  # تخلق قيمة ممتازة
elif economic_spread > 5%:
    score += 15
```

**2. FCF Yield (20 نقطة):**
```python
if fcf_yield > 6%:
    score += 20  # عائد نقدي ممتاز
elif fcf_yield > 3%:
    score += 10
```

**3. Investment Score (20 نقطة):**
```python
if investment_score > 70:
    score += 20
elif investment_score > 50:
    score += 10
```

**4. ROIC (15 نقطة):**
```python
if roic > 15%:
    score += 15  # كفاءة ممتازة
elif roic > 10%:
    score += 8
```

**5. Z-Score (15 نقطة):**
```python
if z_score > 2.99:
    score += 15  # آمن جداً
elif z_score > 1.80:
    score += 7
```

**6. ROE (5 نقطة):**
```python
if roe > 15%:
    score += 5
```

#### **الأحكام النهائية:**

| النتيجة | الحكم | التوصية | الوصف |
|---------|-------|----------|-------|
| **85-100** | جوهرة مخفية 💎 | شراء قوي | استثمار ممتاز بكل المقاييس |
| **70-84** | استثمار ممتاز ⭐ | شراء | جودة عالية، عوائد جيدة |
| **55-69** | استثمار جيد ✓ | احتفاظ/شراء | جودة متوسطة إلى جيدة |
| **40-54** | متوسط الجودة ⚠️ | احتفاظ | مقبول لكن حذر |
| **25-39** | فخ قيمة 🚨 | تجنب | يبدو رخيصاً لكن خطير |
| **< 25** | نمو مفرط ❌ | بيع/تجنب | مخاطر عالية جداً |

#### **مثال عملي:**
```
شركة Apple (مثال افتراضي):
- Economic Spread: 12% (✅ → 25)
- FCF Yield: 4.5% (✅ → 10)
- Investment Score: 82 (✅ → 20)
- ROIC: 28% (✅ → 15)
- Z-Score: 5.2 (✅ → 15)
- ROE: 95% (✅ → 5)

النتيجة: 90 / 100
الحكم: جوهرة مخفية 💎
التوصية: شراء قوي
الترتيب: أفضل من 98% من الشركات
```

## ⚠️ ملاحظات مهمة

### **1. ليس تعلم آلي حقيقي (حالياً):**
- يستخدم **قواعد وخوارزميات مالية** مثبتة
- النتائج **منطقية وقابلة للتفسير**
- يمكن تطويره لاحقاً لاستخدام ML حقيقي

### **2. دقة النتائج:**
- تعتمد على **جودة البيانات**
- كلما كانت البيانات أكمل، كانت النتائج أدق
- يجب دائماً **التحقق يدوياً** من النتائج الحساسة

### **3. الاستخدام الصحيح:**
- هذه أدوات **مساعدة** وليست بديلاً عن التحليل البشري
- استخدمها مع أدوات أخرى (التحليل الأساسي، المقارن، إلخ)
- **لا تتخذ قرارات بناءً على مؤشر واحد فقط**
